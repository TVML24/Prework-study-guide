HTML

Glossary

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • One element may have many attributes
  • A child element is an element nestled inside another element
  • A semantic element describes it's own content and the role it plays on the webpage.
  • An unordered list is bulletpoints/an ordered list is numbered (?)

Elements - Head

  • <Head>- contains metadata (not visible when page is rendered).
  • <!DOCTYPE> (format e.g. html)- statement that indicates to the browser how to render the document.
  • <html lang= "en"> starts our html coding and indicates that it is in the english language.
  • <title> defines the title of the webpage, important for SEOs.
  • <style> defines the CSS styles associated with the webpage.
  • <base> defines the base URL associated with the webpage.
  • <link> connects an external resource to the html document.
  • <meta> defines other meta data, such as; character set, description, keywords, author and viewpoint.

Attributes - Head

  • (inside meta element) charset= "UTF-8" (designates character set used by the page)
  • (inside meta element) http-equiv= "X-UA-Compatible" Content= "IE=edge" (instructs browser of what version of IE page should be rendered as (in this example Edge)).
  • (inside meta element) name="viewport" content="width=device-width, initial-scale=1.0" (designates the viewport, which allows you to control the width and scaling of the viewport, so that it is correctly sized for all devices).

Elements - Body

  • <h1>(through to h6) Makes a heading and represents a level of that heading. e.g. h1 appears at the top of the page and should only appear once, headings need to appear in numberical order.
  • <p> represents paragraph or block of text.
  • <ul> begins an unordered list e.g. bullet points.
  • <ol> begins an ordered list e.g. numerical.
  • <li> begins an item in the list.
  • <img> contains elements and attributes relating to a single image.
  • <br> indicate a line break or empty space.
  • <a> element stands for 'anchor, and represents a link to another page (or the same page)
  • <section> a semantic element that defines a thematic section of content, but does not render.

Attributes - Body

  • (inside img element) src= "www.imagehost.com/example" (tells the webpage where to find the image)
  • (inside img element) alt= "example image" (provides text description of image)
  • (inside img element) width= "(width of image in pixels)" (allows you to set the width of the image)
  • (inside img element) height= "(height of image in pixels)" (allows you to set the height of the image)
  • (inside a element) href= "(www.examplelink.com)">This is an example link! (href points to the url of the link you want to add, followed by the text you want the hyperlink to have).
  • (inside section element) class= "(class name)" id= "(section name)" (here class name relates to the style of the section (CSS?) and id marks the element for Javascript)

CSS

Glossary

  • Inline CSS - to style an element directly into the html file. Inline styles apply to a sepcific html tag, using a style attribute with CSS rule to style a particular page element only. Like in text. e.g. to style a heading <h1 style="color: red;> This will only apply to that instance of h1.
  • Internal Stylesheet - to embed an entire stylesheet directly into the <head> element. This sheet contains rules for the webpage and applies only to this page. You can configure CSS classes and IDs as you would with an external sheet. A single change to the CSS rule will effect all instances of the element. (example to be insterted)
  • External Stylesheet - The most common way to style a website. Preferred because it allows developers to keep all CSS rules in one place which makes design changes easier later on. This require a <link> element inside the <head> element in the html file to the stylesheet. This file must be added to the repo first or the coding will not be reflected.
  • CSS rule - made up of a selector that signifies which element it will apply to (e.g. h1 for all instances of h1), a property (e.g. color:) and a value (e.g. blue;).
  • CSS box model - visual way of representing how modifying different properties will effect the size of the element overall. (example to be included.)
  • (to) designate a class attribute - basically, to make a custom selector for CSS rules.
  • .card - The period preceeding card designates it as a class selector. Any html element assigned to this class will gain the assignment designated to it by the CSS stylesheet. (provide an example)

CSS Properties

  • Margin: 0; - This designates a margin of zero. A margin is the space we want outside of the element.
  • Padding: 0; - This designates a padding of zero. Padding is space left free inside of the element. If you increase padding it will make the whole element bigger.
  • Background-color: blue; - This designates the background colour of the element as blue.
  • colour: blue; - This designates the element should be blue.
  • text-align: center; - This designates that the text alignment should be centered.
  • text-decoration: underline; This designates that the text should be underlined.
  • display: block; - Instructs each image to start on a new line.

CSS Selectors

  • Wild card symbol (*) - declares that all emements on the page will have the following CSS properties.

CSS Values

  • width: 80%; - instructs that the computer should use 80% of the available width.
  • margin: 40px auto; - instructs that the margin should be 40px at the top and bottom and the auto value applied to the left and right.
  • border: 5px solid gray; - instructs that a border should be created with a width of 5px and be solid gray in colour.
  • box-shadow: 5px 10px #888888; - instructs that shadows should be added around the elements frame. In this example 5px is the offset on the x-axis relative to the element, 10px is the offset on the y-axis relative to the element and #888888 is the colour defined by a html hexadecimal code. This affect could also include blur and spread radius, but here they are not defined.

Git Bash/Git Hub/CLI

Glossary

  • SHA - key to access a version or previous version of code/webpage
  • Repository/Repo - A project on github containing all files related to project including other branches.
  • Markdown (.md) - a textfile written in Markdown syntax that contains notes in plain text about the content of a website/code.
  • Issue - Accessed from Git Hub website; way to set tasks, report progress. Open to start and close when done.
  • Staging area - where any changes you have made wait for the next commit.
  • (to) commit - to make a change with a name that tells everyone what the change was. Tip- write commits in the imperative case.
  • push - send a commits to the cloud to await a pull request.
  • pull request- request to merge work done on a separate branch back into the main branch.
  • SSH agent - holds your decrypted keys in memory, so that you can use them.
  • SSH key - An SSH key is an access credential for the secure shell (SSH) network protocol. Generating SSH keys allows developers to interface with certain remote services without having to constantly retype login information.

Git hub/CLI commands

  • pwd - (print working directory) Displays current location in CLI.
  • ls - list of files and folders in current location.
  • cd downloads - (change directory) - in the example it will take us to the downloads folder.
  • mkdir new_folder - make a new directory in current location - in the example it will be called new_folder.
  • touch new_readme.txt - create a new .txt file in the current location - in the example called new_readme.
  • start new_readme.txt - open a .txt file in the current location - in the example called new_readme.txt.
  • eval "$(ssh -agent -s)" - to run SSH agent.
  • git revert S767617 - to revert to a previous SHA - in the example the gibberish is the first 7 characters of the SHA key.
  • git clone XXXX - to clone a repo directory to the current location
  • git remote -v - to connect to remote repo.
  • code . - to open VSC from current directory.
  • git status - view status of branch/commits.
  • git checkout -b feature/startercode - to create a new branch (e.g. checkout a book from the library) - in the example the new branch will be called feature/startercode
  • git add -A - to add all files in the directory to the staging area.
  • git pull origin main - pull files from another branch (to make sure you are using the latest version of the code) - here the branch is called main.
  • git push origin main - push files to another branch (these will wait for pull request) - here the branch is main.
  • git commit -m "bugfix" - to commit all changes in the staging area and leave a message about what has been changed - here the message is bugfix.

File types/Asset types

  • index.html - All of the html coding on a webpage - outiside of asset folder, inside project folder.
  • style.css - An external stylesheet for CSS coding - in assets folder, inside of project folder.
  • readme.ms - Describes basic function of project, not reflected on user side - in parent foplder, outside of project folder.

Git Hub Workflows

SSH key setup

  1. Use ls -al ~/.ssh to check if any existing SSH keys are present
  2. If you see a public key ending in .pub e.g. id_rsa.pub, id_ecdsa.pub, id_ed25519.pub - you can skip generating a new SSH key and use your existing keys to add to the SSH agent.
  3. If you see an error that it doesn't exist, you need to create a new SSH pair
  4. use ssh-keygen -t ed25519 -C "your.email@gmail.com" to create a new SSH key
  5. When you are prompted save the key in your default file location and create a secure passphrase
  6. use eval “$(ssh-agent -s)” to start the SSH agent
  7. Add your private SSH key to the ssh-agent using the ssh-add command
  8. Copy the public SSH key to your clipboard. Be sure not to copy anything else after this point so that your SSH key stays on your clipboard!
  9. Navigate to the GitHub website and log in to your account.
  10. Click on Settings, and then click on SSH and GPG keys.
  11. Add your SSH key.
  12. You may need to add your name using git config --global user.name "yourname"
  13. You may need to add your email using git config --global user.email "YOUR_EMAIL"
  14. In your terminal, enter ssh -T git@github.com. Verify that the fingerprint in the message matches the GitHub’s RSA public key fingerprint. If it does, type yes and you should be connected!

Basic Github Workflow

  1. Find the directory in CLI
  2. Use Code . to open in VCS
  3. Use git status in CLI to check branch info
  4. Use git checkout -b (branch name) to create new feature branch to work on
  5. Confirm move to new branch with git status
  6. Make your changes or add code/files to directory
  7. Use git add -A to add all modifications from the current working branch to the staging area
  8. Use git commit -m "..." to make the commits from the staging area and describe them
  9. Use git pull origin main to confirm our branch is still using the latest version of main
  10. Use git push origin main to push the changes to the remote branch in the cloud.
  11. Visit Repo in browser and complete pull request

To add files/code to project

  1. Unzip files to project folder
  2. Navigate to project folder in CLI
  3. Use code . to open VSC. New files will automatically show. They are now in the staging area ready to be committed and pushed.

JavaScript

Glossary

  • Script - the element in the html document that loads the JavaScript .script file. Must be placed as far down the html coding as possibly; preferably directly before the end of body. e.g. <script src="file destintion/example/main.js> <script>
  • Array - an ordered collection of data that allows multiple values in a single variable. e.g. Var Tim = ["27", "Australian"];. We can then select a value from the array with Tim[0]. Remember that arrayed values are zero-indexed!
  • Object - any kind of JavaScript variable.
  • Operator - a symbol that produces based on two values or variables. e.g. +, -, x, /. Be careful + as it can add mathematically or combine 2 strings (1+1=11).
  • Assignment - a symbol that assigns a variable. e.g. =.
  • Strict Equality - a symbol that tests two values for equality and returns a boolean result (true/false). This symbol is ===.
  • Not - returns the opposite of the value that would otherwise be returned. e.g. myVariable = 3, buuut !(myVariable === 3). The result is false as ! performs the not operation.
  • Does not Equal - tests to see if two values are not equal and returns a boolean result. Opposite of strict equality. e.g. !==.
  • Conditional - a code structure that tests if an expression is true or truthy and executes code based on the result. See examples section.
  • Function - creates a re-usable function that can be used anywhere in the code. See examples section.
  • Event/event handler - listens for activity in the broswer and runs code in response to that activity. e.g. clicking the screen.
  • Dynamic typing - vairables in Javascript may be associated with any value type, or reassigned to a different value type e.g. let pineapple = 117;
  • Weak typing - Javascript allows for implicit type conversion when an operation involves mismatched value types rather than throwing an error. e.g. foo = 42, foo + 1, return 421 (this is the combination of the two strings).
  • Primitive value/immutable - primitive values are values that cannot be changed e.g. null. e.g. changing this value would break JS.
  • Null - expresses that a variable points to no object. See examples for intentional usage of null.
  • Undefined - expresses the a variable points to no value.
  • JavaScript Property - These associate keys with their value. Here keys define the data set and values are the variables that belong to the set. e.g. (Key) FirstName (value) Bugs
  • Data Property - A kind of JavaScript property that associates a certain key with a certain kind of value. e.g. value (can be any kind of value), writable (boolean as to whether the property can be assigned (=)), enumerable (boolean value that indicates whether a property can be used in a for...loop), configurable (a boolean value that indicates whether or not a value can be deleted, modified or be changed to an accessor property).
  • Accessor Property - a key with one of two accessor functions (get and set) to retrieve or store a value.
  • Get - binds an object property to a function that will be called when it is looked up. See example section.
  • Set - can be used to define or store a value.
  • Method - A Javascript method is a property containing a function definition.
  • Argument - Object called arguments work as a local variable available to all function except arrows. This is used to access the parameter that is passed to the function. It can only be accessed inside the function.
  • Truthy/Falsy - A value is considered falsy if it is 0, +-0, On, Null, Undefined, NaN, or a boolean false value. All other values including a boolean true are truthy. Conditional (If statements) trigger conditions only if the value is truthy. See examples.
  • For loop - Uses a predictable pattern of indicies to perform a task on all the items in the array by allowing a single code block to run over and over again. See examples section.

Built-in Functions

  • document.querySelector ("h1"); - selects an element from html. See example section.
  • ExampleVariable.textContent - sets the variables of the text content (e.g.) what is the text. See example section.
  • alert(); - triggers pop-up in browser.
  • return result; instructs the function to return the result of the variable so that it is available to use outside of the variable.
  • document.querySelector("html").addEventListener("click", - instructs the event listener to listen for a click anywhere on the html page.
  • console.log () - outputs the contents of the parentheses to the console.
  • math.floor() - Always rounds down and returns the largest integer less than or equal to a given number.
  • math.random() - returns a pseudo random number that is greater than or equal to zero and less than one.

Miscellaneous

Data Science Concepts

  • Separation of concerns - Making the program modular, so that each section is separate and addresses a different "concern". This makes code easier to maintain and develop further, but requires more initial coding to set up.
  • D.R.Y code - Don't Repeat Yourself. e.g. rather than make separate lines for header and footer in a CSS stylesheet use a comma to combine the two. e.g. use selector header, footer
  • Control Flow - The order in which the computer executes code e.g. in JS it is linear (top to bottom).
